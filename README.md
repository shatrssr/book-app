# Book App (読書管理アプリ - 仮)
- 本を登録・管理するためのWebアプリケーションです。
- 本の情報（価格など）を記録し、読書状況や統計をシンプルに一元管理することを目的としています。
- スマートフォン・iPad・PC 全てのデバイスで快適に操作できるレスポンシブデザインを採用したいです。
- 練習用として作成しています。

## 技術スタック
- フロントエンド: Next.js (React)
- ホスティング: Vercel
- バックエンド & DB: Supabase
  - 認証（Google / メールアドレス+パスワード）
  - データベース
  - APIなしで直接使用可能
  - グラフ用データ取得も対応
- 外部API: Google Books API (本の情報取得)

## ローカルでの開発環境セットアップ
### 1. リポジトリのクローンとパッケージのインストール
```bash
# パッケージのインストール
npm install
```
### 2. 環境変数の設定
- プロジェクトのルートディレクトリで .env.local ファイルを作成し、Supabaseのプロジェクト設定から以下の値を取得して設定します。
`.env.local`の中身：
```bash
NEXT_PUBLIC_SUPABASE_URL=[INSERT SUPABASE PROJECT URL]
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY=[INSERT SUPABASE PROJECT API PUBLISHABLE OR ANON KEY]
```
### 3.開発サーバーの起動
```bash
npm run dev
```
ブラウザで `http://localhost:3000` を開くとアプリケーションを確認できます。
##　主な実装予定機能
- ユーザー登録・ログイン・ログアウト
- 読書状況の管理（未読・読書中・読了・再読希望）
- 読書記録（読んだページ数、読んだ日付、感想など）
- 上記の対象となる本の情報のメモの登録・編集・削除・リンク
- 本との出会い方の登録・編集・削除・リンク
- 読書記録のグラフ表示
## ディレクトリ構成の概要
- app/ - Next.js のルーティング設定
- components/ - 再利用可能なUIコンポーネント群
- utils/supabase/ - Supabase クライアント関連の設定コード
## 今後のロードマップ＆実装予定機能
アプリの機能はフェーズに分けて段階的に実装していく予定です。
### フェーズ1（初期リリース）
- ログイン機能（Googleアカウント / メールアドレス＋パスワード）
- 本の登録機能（ISBN検索 / キーワード検索による情報取得）
### フェーズ2以降（順次追加予定）
- 読書ステータス管理: 「読んでいる」「読みたい」「読み終わった」「もう一回読みたい」のステータス切り替え
- 感想の入力: 読了後の感想メモ（将来的にタイムライン化を予定）
- 日付管理: 登録日、読み始めた日、読了日、読んだ回数の記録機能（任意入力）
- 入手経路の記録: 購入 / 図書館 / 元々持っていた / もらった / その他（＋備考）の記録機能（任意入力）
- 統計のグラフ表示: 今まで読んだページ数の合計グラフ
- 各種集計表示: 入手経路別・作者別・合計金額のデータ集計
- 表示切り替え: 本の一覧をテーブル（表）形式で表示できる機能
